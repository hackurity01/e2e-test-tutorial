# E2E 테스트 모듈화 구조

## 목표

- E2E 테스트를 모듈화하여 재사용성과 유지보수성 향상

## 폴더 구조

### `e2e/pages/`

- 각 페이지별 동작 메서드 정의
- 예: `HomePage`, `LoginPage`, `ProductPage`, `CartPage` 등

### `e2e/fixtures/`

- 테스트용 데이터 파일
- 예: 사용자 정보, 상품 데이터, 테스트 계정 등

### `e2e/helpers/`

- Helper 및 유틸리티 코드
- 예: API 호출, 데이터 초기화, 공통 함수 등

### `e2e/scenarios/`

- 실제 E2E 테스트 케이스
- `pages/`의 메서드를 조합하여 시나리오 작성

## 원칙

- **페이지 객체 패턴**: 각 페이지의 동작을 `pages/`에 캡슐화
- **데이터 분리**: 테스트 데이터는 `fixtures/`에서 관리
- **코드 재사용**: `scenarios/`는 `pages/`의 메서드를 조합하여 작성
- **유틸 분리**: 공통 로직은 `helpers/`에 분리
